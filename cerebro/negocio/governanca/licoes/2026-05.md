# Lições Aprendidas - 15/05/2026

## Problema 1: "Cannot read image.png"

**Sintoma:** Erro ao carregar plataforma.html no navegador

**Causa:** Servidor na porta 8088 (server.py) estava servindo da pasta errada (`context-brain` vs `cerebro/perfil-empreendedor-solo`)

**Solução:** Corrigir server.py para apontar para pasta correta:
```python
FRONTEND_PATH = r'C:\Users\cleit\Desktop\opb-sistema\cerebro\perfil-empreendedor-solo'
```

## Problema 2: Modelos Ollama não funcionam

**Sintoma:** Modelos maiores (llama3, llama3.2-vision) dão erro de memória

**Causa:** RAM disponível insuficiente (~3.4GB)

**Solução:** Usar modelo phi3:mini (2.2GB) que funciona

**Mudanças:**
- `utils/llm_provider.py`: modelo padrão → phi3:mini
- `agents/radagast/analyzer.py`: modelo → phi3:mini

## Lição Chave

> Sempre verificar qual servidor está rodando em qual porta!
> - Porta 5000 = api_server.py (Flask) - API completa
> - Porta 8088 = server.py (simples) - apenas arquivos estáticos

## Lição 4: PWA Dashboard + Sincronia PC-Celular (20/05/2026)

**Contexto:** Criar painel de controle mobile que substitui o terminal (termux.sh) como PWA instalável, e sincronizar trabalho entre PC e celular via GitHub.

**O que foi feito:**
- **Dashboard PWA:** `dashboard.html` com cards para status, iniciar/parar serviços, executar agentes (Radagast, Carrossel, Texto, Consumo), gerenciar Quadro de Avisos
- **Manifest PWA + Service Worker:** `manifest.json` e `sw.js` para instalação como app no celular
- **API endpoints novos:** `/api/servicos/status`, `/api/servicos/parar`, `/api/servicos/reiniciar`, `/api/agentes/executar`
- **Telegram Bot expandido:** Comandos `/carrossel`, `/texto`, `/capavideo`, `/consumo`, `/posicionamento`, `/tarefas`, `/tarefa`, `/concluir`
- **Scripts de sincronia:** `sync.bat` (PC) e `sync.sh` (Termux) com menu enviar/receber
- **`iniciar-dia.bat` atualizado:** Passo 0 pergunta se quer sync antes de subir serviços
- **`.gitattributes`:** Normalização de line endings (CRLF→LF para .sh, CRLF para .bat)
- **`.gitignore`:** `acervo/`, `.env`, `context-brain/` ignorados

**Problemas enfrentados:**
- CRLF no `termux.sh` impede execução no Linux/Android (shebang `#!/bin/bash\r` quebra)
- **Solução:** `.gitattributes` com `*.sh text eol=lf` no repositório
- Flask route ordering: endpoints novos depois do catch-all `/<path:path>` nunca são chamados
- **Solução:** Mover rotas novas para antes da rota catch-all

**Próximos passos:**
- Finalizar Quadro de Avisos com notificações
- Produzir carrosséis reais e testar fluxo completo
- Configurar Radagast como tarefa agendada no Windows

**Regra nova:** Sempre fazer commit + push + deploy na Vercel após cada alteração importante na plataforma.
