# Decisões — Maio 2026

## D-01 — Manter Ollama como provedor LLM padrão

**Data:** 2026-05-22
**Contexto:** Foi analisado o LM Studio como alternativa ao Ollama para o sistema OPB.
- LM Studio instalado e testado (v0.4.13, via winget)
- Modelo SmolLM2-360M-Instruct baixado (258MB)
- SDK Python conecta ao daemon, mas `load_new_instance` trava
- Servidor HTTP (`localhost:1234`) não iniciável via CLI pura — requer clique na GUI
- Ollama atual: tinyllama (~637MB), ~2.3 tok/s (CPU Intel HD 3000, 8GB RAM)
- LM Studio perderia suporte a whisper (transcrição)

**Decisão:** Manter Ollama como provedor padrão.
**Motivo:** Já integrado, funciona de forma confiável (embora lento no hardware atual), tem suporte a whisper. LM Studio adicionaria complexidade sem ganho real de performance dado o gargalo ser o hardware.

**Próximos passos:**
- Se precisar de mais performance, investir em GPU (ex: placa usada com CUDA)
- Se quiser testar modelos diferentes, LM Studio pode ser usado como alternativa manual via GUI

## Hardware Atual
- CPU: Intel Core i5-2400 (2ª geração)
- RAM: 8GB DDR3
- GPU: Intel HD Graphics 3000 (sem aceleração IA)
- SO: Windows
