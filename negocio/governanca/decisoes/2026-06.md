## D-03 — Fonte única de verdade do perfil

**Data:** 2026-06-06
**Origem:** Plano do Cérebro, Fase 1 (Resgate).

> **`perfis/<id>/perfil/*.md` é a fonte única de verdade do perfil. `negocio/governanca/quem-sou.md` é LEGADO (apenas fallback).**

**Estrutura da fonte única (8 arquivos por perfil):**

| Arquivo | Conteúdo esperado |
|---------|-------------------|
| `PERFIL.md` | nome, autores, instagram, telegram, tagline, versiculo, nicho, problema, pilares |
| `REGRAS-ESCRITA.md` | tom_de_voz, valores, regras_escrita, marca_visual |
| `NARRATIVA.md` | missao, visao, origem, objetivos |
| `PUBLICO-ALVO.md` | cliente_ideal, problemas_publico |
| `POSICIONAMENTO.md` | diferencial, proposta_valor, frase_posicionamento, pvu |
| `HABILIDADES.md` | habilidades, resumo |
| `HISTORIAS.md` | historia_profissional, experiencias |
| `COSMOVISAO.md` | valores, crencas (refina REGRAS-ESCRITA) |

**Implicações:**

- `utils/profile_loader.py` faz parse tolerante (sem acento, aliases): `Público Alvo` == `Público-Alvo` == `Cliente Ideal` == `publico_alvo`.
- Multi-perfil nativo (3 perfis: `paz-na-conta`, `toque-de-paz`, `caminho-vida`).
- Quem-sou.md vira LEGADO — manter para histórico, mas novos agentes leem `perfis/<ativo>/perfil/`.

## D-04 — UX gulosa para perfil (quiz guiado)

**Data:** 2026-06-06
**Origem:** Plano do Cérebro, Fase 2.

> **O preenchimento do perfil deve ser opcional, guloso e mobile-first — não um formulário único de 8 seções.**

**Como:**

- 21 perguntas distribuídas em 8 seções, com pesos (essencial=1.0, nice-to-have=0.5).
- Apresentadas **uma por vez** (chat-style, `entrevista.html`), com progress bar e auto-save.
- Schema em `utils/perfil_quiz.py` é profile-aware (faz parse tolerante do MD).
- Botão "Modo Chat" no `formulario.html` (quem prefere formulário completo, mantém).
- Endpoints REST: `GET /api/perfil/quiz/state`, `POST /api/perfil/quiz/save`, `GET /api/perfil/quiz/schema`.
- Usuário pode sair e voltar de onde parou (idempotente, save individual por seção).

**Implicações:**

- Formulário completo (`formulario.html`) **mantém** como alternativa para quem quer ver tudo de uma vez.
- Agentes futuros devem usar `compute_progress()` para detectar perfil incompleto e sugerir continuar o quiz.
